# round-6（2026-07-17）：密钥新增/编辑交互改弹窗 · 全绿

用户反馈：「页顶常驻表单 + 编辑回填页顶」违反常规操作习惯（编辑动作离行远、要跳滚动、模式不直观）。

## 改动（纯前端，无 API 变化）

- 表单整体迁入标准弹窗 `akModal`（复用应用现有 `.overlay`/`.modal-card` 体系，与「新建任务」同款）：列表卡右上「＋ 生成密钥」开弹窗；行内「编辑」开同一弹窗回填（标题「编辑密钥」、确定键「保存修改」、hint 显示目标钥前缀）；取消 / × / 点遮罩关闭即退出编辑态。
- 页面主体只剩密钥列表卡（说明并入 sec-sub）；明文横幅仍在列表上方（生成成功弹窗关闭后展示）。
- 移除旧机件：页顶表单卡、akEditHint / akCancelEditBtn / akSetEditMode / akResetForm。

## 证据（r9 隔离实例）

- API 38/38 PASS（无 API 改动，全量回归）。
- UI 8/8 PASS：U2a 改验「弹窗打开 + 必选报错在弹窗内 + 不关窗」；U2b 验「生成后弹窗关闭 + 明文横幅」；U6 验「编辑态标题/按钮/回填 + 保存后关窗行更新无新明文」；其余行级用例不变。
- 截图复核：列表页干净单卡；弹窗居中、背景模糊遮罩、底部取消/确认，与新建任务观感一致。
- 残留引用检查：grep akCancelEditBtn/akEditHint/akResetForm/akSetEditMode 为空。
