# 任务归档一级页改版

## 目标

把 `#/archive` 从只有工作目录入口的空白目录页，改为可直接了解归档规模、近期归档和跨目录检索的归档工作台；`#/archive/<工作目录>` 仍是原有任务列表与恢复/删除动作入口。

## 现状与问题

- 一级页仅展示目录名与任务数量，无法判断目录最近是否有归档、归档来自哪个运行时，也不能直接找到历史任务。
- 归档数较多时，用户必须逐个进入目录后再搜索，信息密度和定位效率都很低。
- 归档包的恢复、永久删除和详情返回路由已有安全约束，本次不改变后端或任务生命周期。

## 方案

1. 一级页增加总归档数、工作目录数、最近归档时间三项概览。
2. 提供跨目录关键词搜索；命中时直接展示任务结果并保留“进入目录”入口，不改 URL 契约。
3. 目录卡展示任务数、最近归档时间和 provider 摘要；按最近归档排序，将活跃历史优先展示。
4. 无搜索时在一级页展示最近归档任务，供快速查看详情；任务卡仍走既有详情/恢复/删除行为。

## 验证

- `#/archive`：有/无归档、概览、目录卡、近期任务和跨目录搜索。
- `#/archive/<bucket>`：既有 provider/来源/关键字筛选与恢复/删除动作仍可用。
- 通过抽取并编译 `index.html` 内联脚本，及 `node --check platform/public/app.js`、`git diff --check` 做静态校验。
